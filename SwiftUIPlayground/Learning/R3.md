## Challenging work

### VoIP push / missed sessions

Situation: Users missed sessions because VoIP pushes were unreliable.

Task: Find the cause and make sessions reliable.

Action: Added tracking and logging. Built a fallback: if the VoIP push does not arrive, the app still gets the session over the live socket.

Result: Session reliability improved; missed sessions from push failures largely gone.


### KMP + Swift task lifecycle

Situation: iOS app with Kotlin Multiplatform. Kotlin exposed flows; Swift used `Task` and `for await` to observe them.

Task: Presenters and view models should deallocate when the screen is dismissed.

Action: Presenter was not deallocating. It looked like KMP at first, but the issue was Swift: async tasks were not stored or cancelled, so the task kept the flow alive and held the presenter. Added a base view model that tracks flow tasks, cancels them in `deinit`, and calls `presenter.close()` on cleanup.

Result: Presenter deallocated correctly; no stale subscriptions; reusable pattern for KMP on iOS.

Learning: Weak self is not enough; you must cancel async work that observes shared state.
